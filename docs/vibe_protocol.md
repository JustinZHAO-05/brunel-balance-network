# Vibe Coding Protocol

## 目标

验证代理是否能在较少人工干预下，完成一个小型计算神经科学项目的端到端交付：

- 环境搭建
- 建模与参数配置
- 仿真与指标分析
- 图表与结论生成
- 自动验收与记录

## 允许的能力

- 读取本地仓库和官方文档
- 安装 Python 依赖
- 编写与修改源码、配置、测试和文档
- 运行仿真、测试和命令行脚本

## 不允许的捷径

- 直接复制一个现成完整仓库作为主要成果
- 用人工手改结果文件来伪造通过验收
- 绕过 A/B/C/D canonical panels 的官方参数定义

## 人工干预规则

- 允许在依赖兼容性或运行时间过长时调整工程细节
- 不允许修改 Brunel 模型的核心方程和关键参数语义
- 若复现不稳定，只允许调整 `seed`、`sim_time_ms`、扫参分辨率和 quick/poster 的规模

## 验收标准

- `python -m src.reproduce --preset quick` 可以完整跑通
- `artifacts/acceptance.json` 中包含明确的 rule-by-rule 判定
- 图、表、说明文档和代码结构能支持公开展示
